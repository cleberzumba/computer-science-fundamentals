# computer-science-fundamentals

> Computer science fundamentals for engineers — operating systems, concurrency, networking, and data structures.

---

## About this repository

A collection of deep-dives and practical notes on the core computer science concepts that power modern software systems. The goal is to bridge the gap between academic theory and real-world engineering — covering not just *what* these concepts are, but *why* they matter when building production systems.

---

## Topics covered

### Operating Systems
- Processes vs threads
- Memory management and virtual memory
- CPU scheduling
- Context switching
- Inter-process communication (IPC)

### Concurrency & Parallelism
- Concurrency vs parallelism
- Locks, mutexes, and semaphores
- Race conditions and deadlocks
- Synchronization primitives
- Thread pools and async patterns

### Networking
- TCP/IP fundamentals
- HTTP/HTTPS lifecycle
- Sockets and connection handling
- Load balancing strategies
- DNS resolution

### Data Structures & Algorithms
- Arrays, linked lists, stacks, queues
- Trees, heaps, hash tables
- Graphs and traversal algorithms
- Sorting and searching
- Big-O notation and complexity analysis

### Number Systems & Data Representation
- Binary to decimal conversion
- Decimal to binary conversion
- Decimal to hexadecimal conversion
- Binary to hexadecimal conversion

### Computer Architecture
- CPU vs GPU
- Memory hierarchy (registers, cache, RAM, disk)
- Instruction pipelining
- SIMD and vector processing
- Cache coherence

## How to use this repository

Each topic is documented as a standalone Markdown file with:

- **Concept overview** — what it is and why it matters
- **Visual diagrams** — when applicable
- **Code examples** — practical implementations
- **Real-world applications** — how the concept appears in production systems
- **Common pitfalls** — what to watch out for
